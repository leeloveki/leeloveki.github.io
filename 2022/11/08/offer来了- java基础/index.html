集合

Java的集合类主要有4种: List Queue Set Map

  1. List:

    1. ArrayList

    2. LinkedList

    3. Vector

  2. Queue(不重要):

    1. DelayQueue
    2. …

  3. Set

    1. HashSet
    2. LinkedHashSet
    3. TreeSet

  4. Map

    1. HashMap
    2. HashTable
    3. TreeMap
    4. LinkedHashMap
    5. ConcurrentHashMap

数据结构特点

List: 有序, 允许有重复的元素

Set: 是无序, 不允许有重复的元素

Map: 通过键值对保存数据, key(索引)以set集合的形式保存, 不允许有重复的key,

List

List是有序的集合, 有三种常见的实现类

ArrayList: 底层由数组实现, 增删慢, 查询快

ArrayList是使用最普遍的List实现类

对外提供 add remove get方法实现增删查的功能

增删慢的原因: ArrayList是在连续的位置存储元素(数组实现), 导致对ArrayList的元素进行插入删除操作时必须同时对操作元素位置后面的元素进行移动操作. 也可以称为修改代价高

数组特点: 插入删除操作慢, 操作时需要对操作元素后面的元素都进行移动操作, 也称为修改代价高

随机查找(查询)/遍历效率高的原因也是由于数组实现的原因

当ArrayList的元素达到当前数组容量上限时, 会创建一个新的数组并将旧数组的数据复制到新数组中再进行增删操作

Vector

vector与ArrayList类似, 也是基于数组实现, 提供相同的Api操作方法. 不同在于Vector是线程安全的, ArrayList是线程不安全的(效率更高, 应用更广泛)

线程安全: 同一时刻只允许一个线程对集合进行操作, 保证在多线程应用场景下集合的数据一致性, 但是实现线程安全会导致频繁的加锁释放锁操作, 导致IO效率较低

LinkedList

LinkedList基于双向链表实现, 增删操作快, 查询慢, 线程不安全

链表特点: 插入删除操作快(需要操作的元素较少), 查询慢(需要从头遍历所有的元素)

LinkedList还提供了对链表头尾进行操作的功能, 因此可以作为stack或queue使用

Queue

队列结构的实现类在项目开发中比较少使用, 一般在中间件(MQ, kafka)中队列结构的应用较广泛

常见的java队列实现类:

  1. ArrayBlockingQueue
  2. DelayQueue
  3. SynchronousQueue : 线程同步的阻塞队列

Set

Set结构的特点是不允许有重复的元素

根据hashCode和equals方法判断两个元素是否重复

HashCode是对象的内存地址计算

HashSet: HashMap的key底层实现为HashSet

HashSet判断两个元素是否重复:

  1. 根据hashCode和equals方法判断两个元素是否重复
  2. 当且仅当两个方法都返回相同,则视为重复的元素
  3. 当两个方法有一个方法返回不相同时, 则视为非重复元素

TreeSet

TreeSet基于二叉树实现, 并且整棵树的元素位置都是有顺序(升序或降序)的. 因此在添加新元素时需要进行排序, 将新元素插入二叉树中的指定位置

Java中提供的基本类型包装类已经提供了排序方法, 因此可以直接保存到TreeSet中

自定义的对象保存到TreeSet中前必须先实现Comparable接口, 重写接口中的compareTo(对两个对象进行对比)(返回值大于0表示a大于b, 返回值小于0表示a小于b)

LinkedHashSet

LinkedHashSet是在HashSet的基础上使用双链表接口记录元素的顺序

LinkedHashSet是LinkedHashMap存储key的底层实现

使用方法与HashSet类似

Map

HashMap的key去重底层基于HashSet实现, 通过key对应的HashCode来标识或查询数据 , 线程不安全

HashSet特点: 遍历顺序无序, 每次遍历的元素顺序都不相同

HashMap可以继承synchronizedMap接口来实现线程安全, 或者直接使用ConcurrentHashMap代替

HashMap存储结构: 数组+链表/红黑树

数组中的每个元素都是链表或红黑树

默认为链表, 当链表中的元素超过8个后会自动转换为红黑树来提高查询效率

HashMap查找元素的底层原理: 首先通过HashCode定位数组中的元素位置(下标) , 在对应位置的链表或黑红树中查找数据

链表结构查找数据需要遍历整个链表,时间复杂度为O(n)

红黑树的查询效率高, 时间复杂度为O(logN)

HashMap的底层属性:

  1. capacity: 当前的数组容量, 默认为16, 每次扩容时在当前容量的基础上x2
  2. loadFactor: 负载因素, 默认为0.75

ConcurrentHashMap

ConcurrentHashMap与HashMap类似, 但是是线程安全的, 并且在JDK1.7和1.8使用了不同的方式实现线程安全

JDK1.7: 使用分段锁实现线程安全

JDK1.8: 使用Synchronized+CAS 实现对多线程的安全操作, 并且在数据结构中引入了红黑树

HashTable

HashTable属于一种遗留类, 不推荐在正式环境中使用

HashTable的功能与HashMap类似, 但是是线程安全的

TreeMap

TreeMap基于二叉树作为存储的数据结构, 并且实现了SortedMap接口, 保证数据存储按照顺序

默认是按照key的升序排列, 也可以使用自定义的比较器来排序

TreeMap的key必须实现Comparable接口或者使用自定义的比较器, 否则会抛出ClassCastException异常

LinkedHashMap

LinkedHashMap是HashMap的子类, 内部使用链表来保存元素的顺序

使用Iterator遍历LinkedHashMap时会按照链表保存的顺序来遍历

异常的分类和处理

异常的分类

异常是指方法不能正常运行时, 以抛出异常的方式直接退出方法

抛出的异常中一般封装了错误信息和原因, 调用方法的方法可以根据业务情况选择处理异常或者继续往上层调用方抛出异常

异常分类

在java中, 所有的错误或异常都直接或间接继承了Throwable类

Throwable类往下分为Error(错误)和Exception(异常)

Error是由于程序外的原因造成, 比如: 系统死机或内存不足

因此错误是无法被程序处理的, 只能进行记录

常见的错误有: AWTError ThreadDeath

异常又分为运行时异常(RuntimeException)和检查时异常(CheckedException)(又称为编译时异常)

运行时异常: 可以被捕获和处理, 常见的运行时异常有: NullPointException(空指针异常) ClassCastException(类型转换异常) ArrayIndexOutOfBundsException(数组越界异常)

编译时异常: 由编译器在程序编译阶段进行检查, 必须对编译时异常进行捕获和处理, 否则程序无法运行. 常见的编译时异常有 IOException(IO异常) SQLException(SQl语句错误) ClassNotFoundException(类不存在异常)

异常的处理

处理异常可以选择直接抛出异常(不在当前方法处理)或者使用try catch语句捕获处理异常

抛出异常有两种方式: throws和throw(方法内部)

区别:

  1. 位置不同: throws在方法声明, 后面跟异常类 throw在方法内部, 后面跟异常对象
  2. 功能不同: throws声明当前方法运行中可能出现的异常, 将异常类型提供给方法调用者, 而throw直接将异常对象抛出给调用者, throw后面的代码不会被执行(但是finally语句的执行不受异常抛出影响)

使用try catch语句可以捕获异常并处理, 异常信息不会传递给调用者

在正式项目中一般使用抛出异常的方式, 尽量少用捕获异常

反射

java中反射机制一般用于在程序运行时的动态执行场景如: 获取类或对象的信息, 动态调用对象的方法, 动态代理

动态语言: 程序在运行时可以改变自身结构的语言: 比如在创建好的对象中添加新的属性, 在旧方法中注入新的代码

常见的动态编程语言有: JS Ruby Python

C和C++不属于动态语言

由于Java具有反射机制, 因此java被称为半动态语言

通过反射机制, java可以运行时时获取或修改类和对象的属性,方法

java中的对象具有两种类型: 编译时类型, 运行时类型

编译时类型是对象声明时使用的类型

运行时类型是对象在程序运行时赋值时使用的类型

通过运行时类型可以使对象具有多态的表现: 对象根据运行时类型不同, 会具有不同的运行方法(同名, 但是运行时表现不同)

由于运行时类型使对象具有多态表现, 因此程序无法在编译期间确定对象和类的真实信息, 只能在运行时通过反射机制来获取(反射机制的核心功能)

反射API: 用于在运行时动态获取类 对象 或接口的信息, 常用的有4种:

  1. Class类: 获取类的属性或方法等信息
  2. Field类: 获取或设置类的非static变量
  3. Method类: 获取或执行类的方法
  4. Constructor: 类的构造方法

获取对应类的Class对象后, 可以通过调用Class对象来实现反射

获取Class对象的三种方法:

  1. 调用对象的getClass方法

  2. 调用类的static变量class

  3. 调用Class类的static方法forName(“完整类路径”)

Class对象的实例方法有:

  1. getDeclareMethods(): 获取所有的方法

  2. getDeclaredFields(): 获取所有的属性

  3. getDeclaredConstructors(): 获取所有的构造方法

  4. newInstance(): 创建对象

    要求该类具有无参构造器, 如果没有的话需要调用getDeclaredConstructor()获取对应的构造器再用构造器对象调用newInstance方法

通过反射实现动态调用方法的步骤:

  1. 调用getMethods方法, 获取Method对象
  2. 调用Method对象的invoke实例方法

注解Annotation

注解是一个接口, 用于设置关联信息化或元数据(MetaData)

程序可以通过反射获取到注解对象, 再通过注解对象获取注解的信息

大部分的框架都是通过注解实现丰富的扩展功能

元注解负责注解其他注解

Java中提供了四种标准元注解:

  1. @Target :定义注解修饰的对象范围
  2. @Rentention : 定义注解的保留级别(有三种保留级别, 从低到高)
    1. SOURCE: 源代码保留
    2. CLASS: 编译时保留
    3. RUNTIME : 运行时保留
  3. @Documented : 表示javadoc工具需要记录该注解实现文档化
  4. @Inherited: 表示该注解是需要被继承的, 子类会继承父类中的该注解

注解处理器: 使用注解的工具

注解处理的步骤

泛型

泛型是参数化类型, 使用类作为类型

类型擦除: 泛型只用于编译时进行检验, 编译后会将在字节码中的泛型信息进行擦除, 在运行时中无任何作用

泛型可以用于编译时进行类型检测, 防止运行中抛出类型转换异常, 提高代码的安全性和重用性

常见的泛型标记有6种:

泛型标记 全称 说明
E Element 集合中的元素
T Type Java类
K Key 键
V Value 值
N Number 数值类型
? 不确定的java类型

注意这些泛型标记是强制要求, 不能使用其他字母作为标记

泛型可以定义上限和下限

  1. 上限: <? extends T> 表示T的子类或子接口
  2. 下限: <? super T> 表示T的父类或父接口

泛型可以用于定义泛型方法 泛型类 泛型接口 泛型集合

序列化

序列化是指将内存中的对象转换为(二进制)信息, 方便进行传输和存储(持久化)

对象在JVM中的生命周期中为 创建 更新 销毁

持久化后的对象信息可以更方便在网络或者硬盘等介质中进行传输和存储

RPC框架也需要用到序列化

常用的序列化框架有: Kryo ProtoBuf FastJson

Java提供了标准化地序列化API用于序列化对象

  1. 实现java.io.Serializable接口可以使类具有序列化功能 (Serializable接口为标记接口, 没有需要实现的方法)
  2. 序列化和反序列化时需要保持序列化ID一致, 一般用private static final long serialVersionUID定义序列化ID
  3. 序列化不保存静态变量
  4. 父类也需要实现Serializable接口
  5. 使用Tramsient关键字可以防止变量信息被序列化, 反序列化时会进行自动初始化

Java I/O

在java中数据的读取和写入(I/O)以流的形式进行

流根据操作单位分为字节流和字符流两类接口

根据功能可以分为高级流(修饰流)和低级流

字节是计算机操作数据的最小单位, 1byte=8bit

字符流以一个字符(两字节)为单位, 默认用Unicode编码

高级流的实现必须依靠低级流(高级流对低级流处理后的数据进行再处理)

类型 字节流 字符流
输出 OutputStream Writer
输入 InputStream Reader

上面四个类都是抽象类, 有多个实现类

InputStream常用的实现类有:

  1. FileInputStream : 文件输入
  2. ObjectInputStream: 对象输入
  3. ByteArrayInputStream: Byte数组输入
  4. FilterInputStream: 过滤器
  5. StringBufferedInputStream: 缓存字符串流

InputStream的方法有:

方法 作用
read 读取字节数据, 返回值为读取的总字节数, 可以传入byte数组存储数据, 可以传入读取的开始位置和长度(默认为0, 长度8字节)
close 关闭流
available 返回输入流可以读取的位数
skip 从输入流跳过n字节

读取文件流示例

OutputStream的常用实现类与InputStream类似, 只需要将Input替换为Output

常用方法:

方法 作用
write 将字节数据写入输出流
close 关闭输出流
flush 刷新缓冲区

Reader常见的子类有:

  1. CharArrayReader: 将字符数组转为输入
  2. StringReader: 将字符串转换为输入
