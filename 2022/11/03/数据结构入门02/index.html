数据结构入门02

线性数据结构

常见的线性数据结构有: 数组(array) 链表(linked list) 栈(stack) 队列(queue)

数组: 连续存储结构, 存储的元素都是相同类型(相同空间大小)

数组一旦创建, 其空间大小固定, 不可改变(连续存储位置的特性导致的限制)

链表: 每个元素都是对象, 可以动态增加和删除元素, 其线性顺序由指针维护

链表的类型一般有四种,

  1. 单链表: 一个节点只存储下一个节点的指针
  2. 双链表:一个节点存储两个节点的指针(上一个节点和下一个节点)
  3. 环状链表: 链表头尾相连构成一个环状结构, 因此可以从任意节点开始遍历整个链表
  4. 环状双链表:

链表的缺点: 不允许随机访问

在数组中访问第n个元素需要O(1)的时间, 链表中需要O(n)

stack

栈是一种先进后出(FILO)的数据结构, 对栈的添加删除操作都只能在同一端进行

栈的常见操作:

  1. push 入栈
  2. pop 出栈
  3. top 返回栈顶的元素, 但不删除
  4. size 返回栈当前元素总量

栈通常有两种类型: 寄存器栈(固定容量) 内存栈(动态容量)

queue

队列是一种先进先出FIFO的数据结构, 主要有排队(添加新元素到顶部)和出列(删除尾部元素)

enqueue dequeue

队列分类:

  1. 简单队列: 线性队列
  2. 循环队列:
  3. 优先级队列
  4. 双端队列: 可以在两端进行插入或删除操作的特殊队列, 不遵循FIFO规则

分层数据结构

树是典型的分层数据结构

二叉树是一种特殊的树结构, 其每个节点最多只能有两个子节点(左子节点和右子节点)

二叉树的每个节点会包含三个部分: 数据 指向左子节点的指针, 指向右子节点的指针(因此树的遍历只能从父节点开始向子节点进行, 不能逆过来)

二叉树一般有两种方式来遍历: 深度优先遍历, 宽度优先遍历(水平顺序遍历)

深度优先遍历有三种:

  1. 有序遍历: 根节点-左子节点-右子节点
  2. 预序遍历: 左子节点-根节点-右子节点
  3. 后序遍历: 左子节点-右子节点-根节点

二叉树的基本操作:

  1. 插入
  2. 删除
  3. 搜索
  4. 遍历

二叉树的辅助操作:

  1. 计算树的高度
  2. 计算树有多少层
  3. 计算整个树的节点数量

二叉树的应用场景:

  1. 霍夫曼编码树可以用于压缩数据
  2. 二叉树可以实现优先级队列

树遍历(travel tree)

对于树或图有两种常用的遍历算法: DFS BFS

  1. DFS 深度优先搜索 (Depth First Search)
  2. BFS 广度优先搜索(Breadth First Search)

二叉搜索树(BST) 是一种特殊的二叉树

BST要求: 左子节点和右子节点必须小于或等于父节点(对每个节点都有此要求)

BST的常用操作有

  1. 查找最大最小值
  2. 删除特定元素
  3. 插入元素(需要在插入后仍满足BST要求)

辅助操作:

  1. 查找第k小的元素
  2. 判断一个二叉树是否满足BST的要求

bst特性: 同时提供了速度适中的搜索和插入操作

搜索速度: 数组>bst>链表

插入删除操作: 链表>bst>数组

二项堆

二项堆是一种特殊的二叉树

要求:

  1. 必须是完全树(每个层次都被填满)

二项堆的底层一般使用数组实现

散列(Hash)

利用hash存储数据, 可以将搜索 删除 插入等操作的时间复杂度降低到O(1)的程度

平均情况下的时间复杂度为O(1)

散列函数(hash函数):可以将输入的数据转换为固定大小的整数值, 将对应的整数值作为索引, 将数据存储到hash表中

由于hash函数的计算效率很高, 因此根据索引直接取数据的速度非常快

hash插入时不需要考虑索引重复的情况, 因此插入速度非常快

hash函数应该做到以下四点:

  1. 计算速度快
  2. 索引分布的位置均匀
  3. 尽可能降低碰撞概率
  4. 高负载系数

hash表: hash表实质上是一个特殊的数组, 数组元素是指向对应数据的指针

通过hash表可以进行快速的元素搜索

hash碰撞: hash函数计算时有可能发生不同数据计算出相同的索引的情况, 必须对这种情况进行处理, 一般有三种处理方式
