如何介绍自己做的项目:

  1. 对项⽬整体设计的⼀个感受（⾯试官可能会让你画系统的架构图）

  2. 在这个项⽬中你负责了什么、做了什么、担任了什么⻆⾊

  3. 从这个项⽬中你学会了那些东⻄，使⽤到了那些技术，学会了那些新技术的使⽤

  4. 你是如何协调项⽬组成员协同开发的或者在遇到某⼀个棘⼿的问题的时候你是如何解决的⼜或者 说你在这个项⽬⽤了什么技术实现了什么功能⽐如：优化了数据库的设计减少了冗余字段、⽤ redis 做缓存提⾼了访问速度、使⽤消息队列削峰和降流、进⾏了服务拆分并集成了 dubbo 和 nacos 等等

  5. 尽量准备好性能优化的手段介绍, 对项目性能进行压测

JVM: JVM负责执行java字节码, 实现java一次编译到处运行的关键

JVM是一种规范, 不同的公司会有不同的JVM实现

目前通用的JVM为HotSpot VM

JVM对于热点代码(经常被调用的代码)引入了JIT编译器(just-in-time compliation)

JIT编译器属于运行时编译, 并且会将编译后的机器码保存下来, 后面的运行会直接使用

java属于半编译版解释的语言

HotSpot使用惰性评估, 将热点代码交给JIT编译为机器码并保存, 提高运行效率

JDK9中引入了AOT编译模式, 直接将字节码编译为机器码, 与JIT的预热编译不同

由于需要使用动态代理等修改字节码的技术, 因此AOT编码模式不能普遍使用

编译型语言: 将源代码全部编译为机器码, 执行速度快, 开发效率低

解释型语言: 将源代码逐行解释为机器码并执行, 执行速度慢, 开发灵活

成员变量和局部变量的区别：

  1. 语法：成员变量可以被访问控制修饰符修饰
  2. 储存空间： 类变量存储在元空间， 实例变量存储在堆内存， 局部变量存储在栈内存
  3. 生命周期： 类 对象 方法的生命周期
  4. 默认值： 成员变量会被自动初始化， 局部变量不会

通常static变量用final修饰作为常量使用

静态方法只能访问静态成员

重载（overload）， 不同形参，视为不同方法

重写(override)， 同一个方法由于类型不同表现出多态性

重写的要求：

  1. 方法名和参数列表相同 （同一个方法）
  2. 被private/final/static修饰的方法不能被子类重写， static修饰的方法可以被重写定义
  3. 构造方法无法重写（子类访问父类的构造方法需要使用super关键字）
  4. 返回类型相同（如果是引用类型可以为子类）

java有8种基本数据类型:

4种整数型 byte short int long

2种浮点型 float double

1种字符型

1种布尔型

注意java的基本数据类型大小都是固定的, 与平台无关

包装类型只能用equals方法对比， 不能用==

因为包装类型是对象， 引用类型

包装类型的拆装箱操作会影响系统性能， 尽量减少使用

基本类型和包装类型的区别:

  1. 基本类型可以自动初始化为默认值, 引用类型默认为null
  2. 引用类型

拆箱: valueOf()

装箱: 自动装箱(将基本数据类型赋值给包装类型) 手动装箱(强制类型转换)

包装类型的缓存机制:

用于提高性能, 4种整数包装类型都默认提供了[-128,127]的缓存数据

注意缓存池都是针对字面量提供, 通过new创建的对象不会通过缓存池

String字面量也是有缓存池

浮点数据类型由于范围过大所以没有缓存机制提供
