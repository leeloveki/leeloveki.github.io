project 个人中心开发

project 个人中心开发

需求描述

用户可以点击按钮进入个人中心，用户不仅在个人中心可以进行就诊人和住院人的管理，还可以查看门诊缴费额记录、核酸/体检的预约记录等

需要开发的功能:

    • 就诊人管理
      • 添加
        • 一个用户最多绑定五个就诊人
      • 修改
        • 修改详情
          • 先查询详情
          • 校验就诊卡是否存在
          • 校验用户id是否绑定
        • 启用默认
          • 校验用户id是否绑定
          • 校验用户id是否已经绑定默认就诊卡
        • 取消默认
          • 校验用户id是否绑定
          • 校验是否是默认状态
      • 查询详情
        • 校验用户id是否绑定
      • 删除
        • 校验用户id是否绑定
    • 住院人管理
      • 查询列表
        • 根据用户id查询
        • 返回就诊人姓名
      • 查看详情
        • 校验用户id, 就诊卡id是否绑定
        • 返回就诊人信息
        • 返回住院科室信息
      • 通过绑定就诊人绑定住院号
        • (就诊人和住院号绑定只能在后台进行, 用户无法更改就诊人和住院号的绑定关系)
        • 通过用户id查询已绑定的就诊人,筛选出绑定了住院号的就诊人
        • 就住院号和用户id绑定(多对1关系)
      • 用户id和住院号解绑
      • 不提供删除住院号的功能
    • 预约挂号记录
      • 查询列表
        • 默认显示一年内的订单
        • 根据时间搜索
        • 返回科室名称
        • 返回医生名称
        • 返回就诊人名称
      • 查询详情
        • 返回科室信息
        • 返回医生名称
        • 返回就诊人信息
      • 取消预约挂号
        • 校验就诊人id是否绑定
    • 门诊缴费记录
      • 查询列表
        • 默认显示一年内的订单
        • 根据时间搜索
      • 查询详情
    • 门诊充值记录
      • 查询列表
        • 默认显示一年内的订单
        • 根据时间搜索
      • 显示余额
      • 查询详情
    • 住院充值记录
      • 住院号只能通过患者id去查对应的消费 充值 余额记录
      • 查询列表
        • 默认显示一年内的订单
        • 根据时间搜索
      • 查询详情
      • 显示余额

  7. mysql数据库优化

    1. 将数据库的连接超时配置和spring配置进行对齐
    • 其他
      • 消息反馈
      • 消息通知

开发流程

  1. 确定需求描述
  2. 根据原型图确认数据库表字段是否满足需求, 是否需要增加字段 增加表
  3. 根据需求描述和原型图开发后端接口(Controller)
  4. 根据原型图开发前端页面
