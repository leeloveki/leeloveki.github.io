Maven

基本概念:

maven是一个java的项目管理工具, 可以用于项目自动化或依赖管理

命令:

命令 作用
archetype:create 创建项目
clean 清理项目
install 安装项目到本地仓库
compile 编译源代码
test 测试
deploy 远程部署
package 打包项目

maven在项目中的使用:

  1. 拆分项目为多个模块
  2. 选择打包模式
  3. 版本控制
  4. 依赖继承

模块拆分父子项目

子项目

打包模式有三种

pom: 父项目

war: 网页项目

jar: 普通java项目

Git

基本概念:

git是一个版本控制工具, 用于对git仓库进行操作

git仓库分为本地仓库和远程仓库

本地仓库一般是程序员的工作电脑, 远程仓库是公司自建的或者是互联网托管服务商比如github gitee

一般开发中我们先将远程仓库的代码下载到本地仓库中, 在本地仓库中进行项目开发, 再将写好的代码用git上传到远程仓库中

svn也是一个被广泛使用的版本控制工具, 与git的区别: svn是中心化的, git是分布式设计的

命令:

命令 作用
init 初始化仓库
add 路径或文件 添加文件到暂存区中
commit 将缓存区的更改提交到本地仓库
clone 克隆远程仓库到本地
pull 从远程仓库中拉取更改到本地
push 将本地仓库的更改提交到远程仓库
status 查看仓库状态, 有哪些更改
reset 版本回退
rm 删除文件
mv 移动文件
diff 对比缓存区和本地仓库的区别
