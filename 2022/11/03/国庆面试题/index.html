02-你能说下synchronized和lock的区别吗

区别 synchronized Lock
使用 关键字 接口
锁 java对象的monitor锁 自定义锁
无法尝试 通过tryLock判断锁状态
自动释放锁机制 用unlock手动释放锁, 不能自动释放
锁只能被占用资源的线程释放 其他线程也可以用unlock方法释放锁
特性 可重入 不可中断 非公平锁 可重入 可中断 有公平锁和非公平锁
只能自动释放锁导致灵活性较低, 手动释放锁, 适合实现复杂逻辑

总结:synchronized使用灵活, 有自动释放锁机制, 并且锁有独占性 非公平锁

Lock手动释放锁, 可以通过tryLock判断锁状态, 适合实现复杂逻辑 可以选择公平锁和非公平锁

公平锁和非公平锁的区别

公平锁: 所有线程放入队列中排队获取锁, 保证顺序

优点: 所有线程都能获取到资源

缺点: 吞吐量低, 同一时刻只有一个线程允许, 性能较低

非公平锁不保证顺序, 性能较高, 但是会出现有些线程

05-post请求和get请求的区别

GET和POST的区别

GET POST
reload不影响服务器 reload会导致数据被重复发送
可以被浏览器缓存 从不缓存
用户在地址栏输入url会默认使用GET方法
会保存到浏览器历史记录 不会被历史记录保存
数据有长度限制(因为get请求的数据是直接加到url上, url长度限制为2048) 数据无长度限制
用于请求数据返回 用于发送数据到服务器
数据必须是使用ASCII编码的字符 可以发送二进制文件
明文使用url发送, 不安全 发送数据不会显示在url中, 较安全

总结: get不安全, 请求参数是编码到url里面发送, 有长度限制, 并且必须是ascii编码的字符, post比较安全, 可以发送二进制文件, 没有长度限制

Spring bean生命周期

分为4个阶段: 实例化–> 属性填充 –> 初始化 –> 销毁

细化步骤:

  1. 实例化 ：通过反射构建相关的实例对象，并分配内存。
  2. 属性填充(依赖注入Setter)：设置相关的对象的基本属性和相关的信息方法调用。
  3. 检查Bean是否实现Spring Awareness相关接口
    1. BeanNameAware接口会调用setBeanName方法, 设置bean的id
    2. BeanFactoryAware接口会调用setBeanFactory()方法,设置工厂
    3. ApplicationContextAware接口调用setApplicationContext方法
  4. 初始化
    1. BeanPostProcessor接口会调用postProcessBeforeInitialization方法
    2. InitializingBean接口会调用afterPropertiesSet()
    3. 调用通过init-method属性配置的初始化方法
    4. postAfterInitialization方法
  5. 如果关联了BeanPostProcessor接口，将会调用(Object obj, String s)方法
  6. 当Bean不再使用的时候，会进行销毁。如果Bean实现了DisposableBean,则调用它所实现的afterPropertiesSet()方法。
  7. 调用通过destroy-method属性配置的销毁方法.
